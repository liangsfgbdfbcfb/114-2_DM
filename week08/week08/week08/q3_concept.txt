Q1：為什麼交叉驗證比單次 train_test_split 更能反映模型的真實效能？
如果只做一次 train_test_split，可能會遇到什麼問題？

A1：

交叉驗證（Cross Validation）能夠更準確地評估模型效能，因為它會將資料分成多個部分，並輪流使用不同資料作為測試集進行驗證。以 5-Fold Cross Validation 為例，資料會被分成五份，每次使用其中一份作為驗證資料，其餘四份作為訓練資料，最後再將五次結果取平均。

這種方法的優點是每筆資料都有機會參與訓練與測試，因此能夠降低資料切分所造成的誤差，讓評估結果更具代表性。相較之下，如果只使用一次 train_test_split，測試結果可能受到資料分配方式影響。當測試集剛好過於簡單或過於困難時，模型評估結果就可能失真，無法真實反映模型的泛化能力。

Q2：GridSearchCV 中的 cv=5 代表什麼意思？
如果參數搜尋空間有 4 個 C 值 × 4 個 gamma 值 × 2 個 kernel，
加上 cv=5，總共需要訓練幾次模型？請列出計算過程。

A2：

在 GridSearchCV 中，cv=5 代表使用五折交叉驗證。也就是說，每組超參數都需要進行五次模型訓練與驗證，最後再計算平均分數作為該組參數的評估結果。

首先計算參數組合數量：

4 個 C 值 × 4 個 gamma 值 × 2 個 kernel

= 32 組參數組合

由於每組參數都要執行 5 次交叉驗證，因此總訓練次數為：

32 × 5 = 160

因此 GridSearchCV 在此設定下總共需要訓練 160 次模型，才能完成所有參數搜尋與評估工作。

Q3：SVM 的參數 C 控制什麼？C 值很大和很小分別會造成什麼效果？
這跟過擬合／欠擬合有什麼關係？

A3：

SVM 中的 C 參數主要用來控制模型對分類錯誤的容忍程度，也就是在「維持較大間隔（Margin）」與「降低分類錯誤」之間進行平衡。

當 C 值設定較大時，模型會盡量將所有訓練資料正確分類，即使必須建立較複雜的決策邊界也在所不惜。雖然訓練集表現通常很好，但容易受到雜訊或離群值影響，進而產生過擬合問題，使模型在新資料上的表現下降。

相反地，當 C 值設定較小時，模型允許部分資料被錯誤分類，並傾向建立較寬鬆且簡單的決策邊界。這有助於提升泛化能力，但若 C 過小，模型可能無法充分學習資料特徵，導致訓練集與測試集表現都不理想，也就是欠擬合的情況。

因此在實務上通常會透過 Grid Search 或交叉驗證尋找最適合的 C 值，以取得最佳的分類效果。
